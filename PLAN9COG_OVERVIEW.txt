KEY FEATURES
=============================================================================

AtomSpace - Hypergraph Knowledge Representation:
  • Nodes: ConceptNode, PredicateNode, etc.
  • Links: InheritanceLink, SimilarityLink, etc.
  • Truth Values: Probabilistic (strength, confidence, count)
  • Attention Values: Economic importance (STI, LTI, VLTI)
  • Operations: Create, find, query, pattern match, serialize

PLN - Probabilistic Logic Networks:
  • Deduction: A→B, B→C ⇒ A→C
  • Induction: A→B ⇒ B→A (with probability)
  • Abduction: A→B, B→C ⇒ C→A (with probability)
  • Revision: Combine multiple evidence sources
  • Boolean: AND, OR, NOT with uncertainty

ECAN - Economic Attention Network:
  • STI Management: Short-term importance allocation
  • Attentional Focus: Top-N atoms by importance
  • Attention Spreading: Propagate through graph
  • Attention Decay: Gradual reduction
  • Resource Allocation: Economic model for limited attention

Distributed Cognitive Processing:
  • MachSpace: Distributed hypergraph across systems
  • Cognitive Fusion Reactor: Multi-process task processing
  • 9P Integration: Standard Plan 9 protocol
  • Network Transparency: Remote atoms appear local

  $ cogctl atom list               # List atoms

  $ cogctl ecan focus              # Show focus
  $ cogdemo                        # Run demo

Programming:
  #include <u.h>
  #include <libc.h>
  #include <plan9cog.h>
  
  Plan9Cog *p9c = plan9coginit();
  Atom *cat = atomcreate(p9c->atomspace, ConceptNode, "cat");
  TruthValue tv = {0.9, 0.8, 10};
  atomsettruth(cat, tv);
  plan9cogfree(p9c);

PERFORMANCE CHARACTERISTICS
=============================================================================

AtomSpace:
  • Create atom: O(1) amortized
  • Find atom: O(n) linear search (needs hash table)
  • Delete atom: O(n)
  • Query: O(n) with predicate
  • Pattern match: O(n*m)

PLN:
  • Single rule: O(1)
  • Forward chaining: O(r*d) where r=rules, d=depth
  • Backward chaining: O(r*d)
  • URE chaining: O(r*i) where i=iterations

ECAN:
  • Update: O(n log n) for sorting
  • Allocate: O(1)
  • Spread: O(m) where m=incoming links
  • Decay: O(n)

Thread Safety: Lock-based with fine-grained locking
